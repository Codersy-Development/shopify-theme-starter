# JavaScript / Web Components Cleanup — Design

**Date:** 2026-05-10
**Sub-project:** 2 of 6 in the broader theme cleanup effort
**Status:** Draft (pending user review)

## Context

Sub-project 1 (build & tooling) merged on 2026-05-10 via PR #1, adding Prettier, Theme Check, and npm scripts. This sub-project tackles the 11 JavaScript files in `/assets/` (excluding the generated `application.css`).

The starter theme is cloned for multiple downstream projects. Anything inconsistent here propagates. The current state mixes a partial pattern (an unused `Component` base class) with three independent re-implementations of focus-trap, body-scroll-lock, and escape-to-close behavior. One file (`product-card.js`) does not follow the custom-element pattern at all, and `predictive-search.js` builds result HTML by string concatenation with inconsistent escaping.

Theme Check has no JavaScript-related findings — the `paginate` `UndefinedObject` belongs to sub-project 3 (Liquid sections).

## Current State

`assets/` contents:

- `accordion.js` — `<accordion-group>` extends `HTMLElement` directly.
- `application.css` — generated by Tailwind. Out of scope.
- `cart-drawer.js` — `<cart-drawer>` extends `HTMLElement`. Owns its own focus trap, escape handling, body-scroll-lock, return-focus.
- `cart-icon.js` — `<cart-icon>` extends `HTMLElement`.
- `collection-filters.js` — `<collection-filters>` extends `HTMLElement`. ~180 lines covering: drawer toggle, form auto-submit with debounce, custom range slider with mouse + touch drag handlers, grid switcher with `localStorage` persistence.
- `component.js` — `Component extends HTMLElement` with `setup()`, `$()`, `$$()` helpers. Defined and exported but **not used by any other file**.
- `header-drawer.js` — `<header-drawer>` extends `HTMLElement`. Duplicate of cart-drawer's focus trap, escape handling, body-scroll-lock.
- `header-nav.js` — `<header-nav>` extends `HTMLElement`. Hover/click dropdown.
- `predictive-search.js` — `<predictive-search>` extends `HTMLElement`. Result HTML built via string concatenation; `#escape()` is called on titles but **not** on URLs, prices, image src, or alt.
- `product-card.js` — **not a custom element**. Global `document` click listener delegating off `[data-product-card]`.
- `product-form.js` — `<product-form>` extends `HTMLElement`.
- `product-gallery.js` — `<product-gallery>` extends `HTMLElement`.

Loaded via `snippets/scripts.liquid` using an importmap and per-template `<script type="module">` tags.

## Goals

- Adopt the existing-but-unused `Component` base class consistently across all custom elements.
- Eliminate duplicated open/close + body-scroll-lock + escape-to-close + focus-trap logic by introducing a `Drawer` subclass of `Component`.
- Split `collection-filters.js` into three focused custom elements: `<collection-filters>` (orchestrator), `<price-range-slider>` (drag + value sync), `<grid-switcher>` (column toggle + `localStorage`).
- Convert `product-card.js` from a global click listener into a `<product-card>` custom element. Update `snippets/product-card.liquid` to use the new wrapper.
- Fix the inconsistent escaping in `predictive-search.js` — apply `#escape()` to every interpolated value (titles, prices, alt, URLs, query in "View all" link).
- Fix `price-range-slider`'s missing `change`-event dispatch so desktop auto-submit fires when the user finishes dragging a thumb.
- Normalize style across all `/assets/*.js`: `setup()` instead of inline `connectedCallback`, consistent private fields, consistent exports, consistent file-header JSDoc.

## Non-Goals

- No ESLint, no test runner, no JS bundler (consistent with sub-project 1's decisions).
- No JSDoc-as-types or TypeScript.
- No changes to existing event names: `cart:open`, `cart:updated`, `cart:refresh`. These form the established API between components.
- No refactor of `CartDrawer.refresh()`'s section-fetch + DOM-replace logic. Works as-is.
- No changes to `shopify.theme.toml`, `.shopifyignore`, or `package.json` scripts.
- No fixes to the existing 34 errors / 20 warnings reported by `npm run theme:check`. Those belong to sub-projects 3 and 4.
- No changes to CSS class names referenced from JS (e.g. `is-open`, `is-active`, `md:grid-cols-N`). CSS is sub-project 6.

### `Component` (existing, unchanged contract)

- `connectedCallback()` calls `setup()`. Subclasses override `setup()` instead of `connectedCallback()`.
- Helpers: `$(selector)`, `$$(selector)`.

The class is not modified. We just start using it.

### `Drawer extends Component` (new)

- In `setup()`, reads `[data-panel]` and `[data-overlay]` from `this`.
- `get isOpen()` — reads the `is-open` class on the panel.
- `open()` — adds `is-open` to panel and overlay, locks `document.body.style.overflow`, captures `document.activeElement` for return focus, focuses the first `[data-close]` after a `FOCUS_DELAY_MS` (350) timeout to wait out the CSS transition.
- `close()` — reverses `open()` and returns focus to the captured element.
- Binds one document-level `keydown` listener gated on `isOpen`: Escape closes; Tab calls `#trapFocus(e)` against the panel.
- Binds overlay click → `close()`.
- Binds every `[data-close]` inside `this` → `close()`.
- Constants: `Drawer.FOCUS_DELAY_MS = 350`.

**Subclass contract:** subclasses always call `super.setup()` first. The base class establishes panel, overlay, listeners, and return-focus state before subclass logic runs. This rule is documented in the JSDoc of both `component.js` and `drawer.js`.

### `collection-filters` orchestration

- `<collection-filters>` keeps responsibility for: form auto-submit on change (desktop), the toggle button (Show/Hide on desktop, Open/Close drawer on mobile), and the desktop "Show Filters" / "Hide Filters" label.
- For mobile drawer mode, the orchestrator instantiates a private `Drawer` controller against `[data-filters]` + `[data-filter-overlay]`. To make `Drawer` re-usable as a controller (not just a custom element), `Drawer` exposes a static `Drawer.controllerFor(panel, overlay, host)` that returns an object with `open()`, `close()`, `isOpen`. The custom-element form of `Drawer` is a thin wrapper around the same controller. This avoids forcing `<collection-filters>` to re-implement drawer behavior while still keeping the abstraction in one place.
- `<price-range-slider>` and `<grid-switcher>` are independent custom elements rendered inside the filters markup. They communicate with the form via standard DOM events (firing native `change` on the underlying inputs after value updates, so the orchestrator's existing form-`change` listener picks them up — this is the bug fix for desktop auto-submit on slider drag).

### `product-card`

- `setup()` binds a click listener scoped to `this`. No document-level delegation. No `closest("[data-product-card]")` walk.
- `data-swatch`, `data-image-src`, `data-image-srcset`, `data-url` attributes on swatch buttons stay the same.
- `snippets/product-card.liquid` changes its outer `<div data-product-card>...</div>` to `<product-card>...</product-card>`. The `data-product-card` attribute is retained on the new element to keep any CSS selectors (`[data-product-card]`) matching until sub-project 6 reviews CSS holistically.

### `predictive-search`

- Existing `#escape()` private method is the only escaping mechanism. Apply it consistently:
  - product `title`, `url`, `price`
  - image `src`, image `alt`
  - page `title`, `url`
  - article `title`, `url`
  - the `query` in the "View all results" link (`encodeURIComponent` for URL component, then `#escape` for HTML attribute context)
- No structural change. Result HTML is still built via string concatenation.
- "Start typing to search..." and "Searching..." HTML strings move to `static` constants on the class (`PredictiveSearch.EMPTY_HTML`, `PredictiveSearch.LOADING_HTML`).

## Testing & Verification

No test runner is added. Verification is manual against `shopify theme dev`, scoped to surfaces touched by this pass.

### Smoke checklist (run on the development store)

1. **Drawer base — cart-drawer.** Click the cart icon → drawer opens, body scroll locks, focus moves into the drawer. Tab cycles within the panel (focus trap). Escape closes; focus returns to the cart icon. Overlay click closes. Quantity +/- still updates the cart and refreshes the drawer markup.
2. **Drawer base — header-drawer.** Hamburger button opens, `aria-expanded` toggles, focus trap and escape behavior matches cart-drawer. Return focus lands on the hamburger button.
3. **collection-filters orchestrator.** On desktop ≥768 px: toggle button hides/shows the sidebar; label flips between "Show Filters" / "Hide Filters". Changing a checkbox auto-submits. Number inputs auto-submit after 800 ms debounce. On mobile <768 px: toggle opens the drawer (uses `Drawer` controller — same trap/escape/overlay behavior). Apply / Clear continue working.
4. **price-range-slider (with fix).** Drag a thumb on desktop — form auto-submits after the drag completes (new behavior). Min/max labels update during drag. Mobile: drag works; submission waits for Apply (existing behavior, gated by mdQuery).
5. **grid-switcher.** Click 3-col / 4-col → grid changes, active state moves to the clicked button, value persists across reload.
6. **product-card.** Click a swatch on a collection grid → image swaps, srcset swaps, all links in that card point at the variant URL, active ring moves to the clicked swatch. Other cards on the page remain unaffected.
7. **predictive-search.** Type a query → results render. Inspect rendered HTML for any non-escaped value in titles, URLs, prices, alt. Spot-check by searching for a product whose title or URL contains characters like `<` or `&`.
8. **product-form.** Submit add-to-cart → button shows "Adding...", drawer opens, count updates, screen-reader status fires. On forced failure (block the request in devtools), button shows "Error — try again" and re-enables after 2 s.
9. **No-regression spot check.** Open product page (gallery + variant sync), home page (no JS errors), 404 page.

## Risks

- **Drawer subclass refactor changes nothing externally, but `super.setup()` ordering matters.** A subclass that wires its own listeners must call `super.setup()` first so `[data-panel]` / `[data-overlay]` are set before the custom listeners reference them. *Mitigation:* a single ordering rule documented in `drawer.js` and `component.js` JSDoc.
- **`<product-card>` upgrade timing.** Custom elements upgrade asynchronously. Layout could shift if CSS depends on `[data-product-card]` selectors and behaves differently against `<product-card>`. *Mitigation:* keep `data-product-card` attribute on the new element; defer CSS selector cleanup to sub-project 6.
- **`price-range-slider` firing `change` events on inputs is a deliberate behavior change.** Desktop now auto-submits when the user releases a slider thumb. *Mitigation:* called out in goals as a fix; verified manually in the smoke checklist.
- **Importmap entries point at hashed asset URLs (`asset_url`).** Adding three new entries grows `snippets/scripts.liquid`. No runtime risk.
- **Unconditional `drawer.js` script tag adds one HTTP request even on pages without drawers.** Small and cached. Acceptable for a starter that does not yet have a JS bundler.
