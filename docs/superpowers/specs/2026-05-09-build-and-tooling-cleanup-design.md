# Build & Tooling Cleanup — Design

**Date:** 2026-05-09
**Sub-project:** 1 of 6 in the broader theme cleanup effort
**Status:** Approved

## Context

This is a Shopify starter theme used as the base for multiple downstream projects. The user requested a "full cleanup pass" of the theme. Because that scope spans build tooling, ~12 JS files, Tailwind/CSS, 23 sections, 11 snippets, layouts, templates, configs, and i18n, the work was decomposed into six sub-projects:

1. **Build & tooling** ← this spec
2. JavaScript / web components
3. Liquid sections
4. Liquid snippets & layout
5. Templates & config
6. CSS / Tailwind 4

Build & tooling is intentionally first because adding a linter and formatter immediately surfaces real issues across all other layers, sharpening subsequent passes.

## Current State

- `package.json` only builds CSS via Tailwind 4 CLI; no linter, no formatter, no Theme Check, no JS bundler
- `shopify.theme.toml` has dev and prod environments pointing to the same store (intentional for a starter — left alone)
- No `.editorconfig`, no Prettier config, no ESLint config, no `.theme-check.yml`
- `.gitignore` is minimal (3 lines)
- 12 JS files in `/assets/` are loaded individually
- Shopify CLI 3.90.1 is available locally, which bundles Theme Check

## Goals

- Add code formatting (Prettier + `@shopify/prettier-plugin-liquid`)
- Add Liquid linting (Shopify Theme Check, run via the existing Shopify CLI)
- Wire both into npm scripts
- Do not change runtime behavior

## Non-Goals

- ESLint (declined)
- Husky / lint-staged pre-commit hooks (declined)
- JS bundling (declined — keep loose ES modules in `/assets/`)
- Fixing issues that Theme Check surfaces — captured as input for sub-project 2/3
- Touching `shopify.theme.toml`
- Touching `.shopifyignore` (looks correct as-is)

## Changes

### 1. Prettier

Add devDependencies:

- `prettier@^3.3`
- `@shopify/prettier-plugin-liquid@^1.9`

Files to add:

- `.prettierrc.json` — plugin registered; opinionated defaults: `printWidth: 100`, `singleQuote: false`, `trailingComma: "all"`, `tabWidth: 2`. Liquid-specific options will use plugin defaults until a specific need arises.
- `.prettierignore` — exclude:
  - `node_modules/`
  - `assets/application.css` (generated by Tailwind)
  - `package-lock.json`
  - `.shopify/`
  - `locales/*.json` (Shopify-owned format)

### 2. Shopify Theme Check

No npm dependency — uses the Shopify CLI already required for `shopify theme dev`.

- `.theme-check.yml` — start with the default ruleset enabled. Override known-noisy checks (e.g. `MissingTemplate`) to severity `warning` so they don't fail CI later.
- During implementation, run `shopify theme check` and capture findings into a separate notes file (e.g. `docs/superpowers/notes/2026-05-09-theme-check-findings.md`) so they can feed the next sub-projects without being lost.

### 3. package.json — scripts

Add:

- `format` → `prettier --write .`
- `format:check` → `prettier --check .`
- `theme:check` → `shopify theme check`
- `lint` → `npm run format:check && npm run theme:check`

Existing scripts (`build:css`, `watch:css`, `dev`, `deploy`) remain untouched.

### 4. One-time formatting commit

Running `npm run format` for the first time will touch most files (whitespace, quote style, Liquid tag spacing). This is unavoidable.

Mitigations:

- Land Prettier setup and the formatting run in two separate commits so review is sane.
- Add `.git-blame-ignore-revs` containing the SHA of the formatting-run commit so `git blame` skips it. Document in a one-line comment in the file.

### 5. `.gitignore` additions

Append (small, only what's defensible for a JS/Shopify project):

- `.env`
- `.env.local`
- `*.log`

Do not add editor-specific ignores (`.vscode/`, `.idea/`) — those belong in user-global gitignore.

## Risks

- **Noisy diff from first format run.** Mitigated by isolated commit + `.git-blame-ignore-revs`.
- **Theme Check surfaces findings that look like real bugs.** Out of scope for this pass; documented in the findings notes file for sub-projects 2–6.
- **Prettier + `@shopify/prettier-plugin-liquid` version drift.** Pinned with caret ranges; version bumps handled in normal dependency maintenance.

None as of approval.

## Acceptance Criteria

- `npm install` succeeds with the new devDependencies.
- `npm run format:check` passes (after the formatting commit).
- `npm run theme:check` runs. It exits 1 today because the captured Theme Check report contains real errors that future sub-projects will fix; see [docs/superpowers/notes/2026-05-09-theme-check-findings.md](../notes/2026-05-09-theme-check-findings.md). The exit will return to 0 once those issues are addressed.
- `npm run lint` runs both checks in sequence (will inherit `theme:check`'s exit code until the findings are addressed).
- Two new commits exist: one introducing tooling, one applying the format run.
- `.git-blame-ignore-revs` exists and contains the formatting-run SHA.
- Theme Check findings are captured in `docs/superpowers/notes/2026-05-09-theme-check-findings.md`.
- No runtime files (`*.liquid` logic, JS behavior, generated CSS) have semantic changes — only formatting.
